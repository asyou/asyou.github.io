Mac安装PHP以及自带Apache配置文件httpd.conf配置参考

2023-02-02

苹果MacOS12之前的系统版本内置了python2和php7，作为使用python和php的开发者来说非常方便。

但是在MacOS12中php模块被移除了，需要自行安装。本文记录一下在12系统上安装php7.4，以及本机的配置apache相关配置、虚拟机多网站等。

一、安装Homebrew

/bin/zsh -c "$(curl -fsSL https://gitee.com/cunkai/HomebrewCN/raw/master/Homebrew.sh)"

安装完后执行一下更新：

brew update

二、安装php7.4

brew install php@7.4

1、注意，安装完成后会有echo xxx和export xxx，逐一复制并执行确保后续使用php相关命令正常。

2、检查一下php版本，输入命令：

wangwei@iMac ~ % php -v
PHP 7.4.32 (cli) (built: Sep 29 2022 11:04:22) ( NTS )
Copyright (c) The PHP Group
Zend Engine v3.4.0, Copyright (c) Zend Technologies
    with Zend OPcache v7.4.32, Copyright (c), by Zend Technologies

如显示上述信息，则安装正常。

三、给php模块签名

这一步非常重要，由于MacOS12系统的安全限制升级，php安装完后并不能直接配置到httpd.conf中，会报错！

打开系统工具，找到“钥匙串访问”并打开，在菜单栏中点击“钥匙串访问”找到“证书助理”，从中选择“创建证书颁发机构”，弹出框如下：

名称和邮件必填，但内容随意。

创建好后，在“我的证书”中找到刚刚创建的证书，双击打开，在“信任”那一栏中全部选择“始终信任”。

三、修改httpd.conf

1、打开终端，输入命令：

wangwei@iMac ~ % sudo vi /etc/apache2/httpd.conf

回车后输入密码进入编辑，找到如下几项，把前面的#号删除

LoadModule vhost_alias_module libexec/apache2/mod_vhost_alias.so
LoadModule rewrite_module libexec/apache2/mod_rewrite.so
Include /private/etc/apache2/extra/httpd-vhosts.conf

第一个和第三个表示启用vhost虚拟机，第二个表示启用php重写，

2、添加php模块，找到LoadModule结尾处，添加如下

#PHP was deprecated in macOS 11 and removed from macOS 12
LoadModule php7_module /usr/local/opt/php@7.4/lib/httpd/modules/libphp7.so "iMacCA"

这个就是上面安装的php模块，后面的“iMacCA”是刚刚创建的证书名称，确保名称一致且正确。

3、其他需要修改的地方

<Directory />
    AllowOverride None
    Require all granted #把denied改成granted
</Directory>

DocumentRoot “website document” #改成自己的网站根目录，比如/Users/wangwei/wwwroot/
<Directory “website document”>  #跟上面一样
    AllowOverride All #把None改成All
</Directory>

<Directory "/Users/wangwei/wwwroot">
    AllowOverride All #把None改成All，解决重写无效问题
</Directory>

修改完后，按ESC，同时按下Shift+:并输入wq保存。

四、修改虚拟机配置，输入命令：

wangwei@iMac ~ % sudo vi /etc/apache2/extra/httpd-vhosts.conf

进入后，里面有一个默认的示例Vhost配置，直接修改或复制一份，然后编辑：

<VirtualHost *:80>
    ServerAdmin webmaster@dummy-host.example.com
    DocumentRoot "/Users/wangwei/wwwroot/www.xxx.com/public" #网站目录
    ServerName www.xxx.cn #主域名 
    ServerAlias *.xxx.cn  #二级域名
    ErrorLog "/private/var/log/apache2/dummy-host.example.com-error_log"
    CustomLog "/private/var/log/apache2/dummy-host.example.com-access_log" common
</VirtualHost>

保存并退出。

五、修改hosts，输入命令：

wangwei@iMac ~ % sudo vi /etc/hosts

添加下面一行：

127.0.0.1    www.xxx.cn m.xxx.cn api.xxx.cn

多个二级域名用空格隔开，注意此处跟虚拟机里配置的本地域名都是以.cn结尾，假设线上的域名是.com结尾，这样写是为了避免与线上域名冲突（本地配置的域名不能跟线上的一致，否则本地调试会有问题）。

六、把网站文件复制到/Users/wangwei/wwwroot/目录中，并修改名称为www.xxx.com，比如下载ThinkPHP官方包放进去。

七、重启apache，输入命令：

wangwei@iMac ~ % sudo apachectl restart

八、打开浏览器，地址栏输入www.xxx.cn，开始享受美好的开发之旅吧～


其他说明：

1、网上的技术教程清一色都极其简陋，都是用localhost访问本地项目，但这样就没法在本地调试多个项目了。而按照本文的方法可以在本地添加多个网站，只需要重复上述第四到第八步骤即可；

2、建议使用项目域名作为网站目录名称，并且后缀跟线上区分出来，比如线上是www.abc.com，则本地域名配置成www.abc.cn，这样本地调试、线上预览两不耽误。
